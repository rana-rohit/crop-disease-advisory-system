Problem Statement

Why This Matters

Agricultural crop diseases cause an estimated 20–40% loss in global food production annually. Early, accurate diagnosis is the single most effective intervention — yet access to trained agronomists is severely limited in rural regions. CropSense AI addresses this gap with an automated, explainable, and actionable diagnostic tool accessible from any device.

🌾
Food Security
Crop diseases threaten global food supply. AI-assisted early detection reduces losses before they escalate to field-wide impact.
🔬
Expert Scarcity
Agronomists are scarce and expensive. A smartphone-accessible AI diagnostic removes the bottleneck of expert availability.
🤝
Trust in AI
Black-box AI decisions are rejected by practitioners. Grad-CAM explainability builds trust by showing what the model is looking at.

Technology Stack

How It's Built

MobileNetV2
Deep Learning Backbone
Lightweight inverted residual architecture pretrained on ImageNet. Top classification layers replaced and fine-tuned on PlantVillage. Delivers <1s inference at 224×224 input resolution with <4MB model size.
Grad-CAM
Explainability · XAI
Gradient-weighted Class Activation Mapping computes the gradient of the predicted class score with respect to the last convolutional layer's feature maps — producing a spatial heatmap without modifying the model architecture.
Flask REST API
Backend · Python
Python Flask backend exposes a single prediction endpoint accepting multipart image uploads. Returns structured JSON containing disease label, confidence, top-3 predictions, advisory content, Grad-CAM overlay, and model metadata.
Vanilla Frontend
HTML · CSS · JavaScript
Pure HTML, CSS, and JavaScript — no frameworks or build tools required. Designed for direct deployment on any static host. The CSS design system uses custom properties for consistent theming across all pages.
Inference Pipeline

From Image to Advisory

1
Image Upload & Validation
User uploads a JPG, PNG, or WEBP crop leaf image via the web interface. The frontend validates MIME type and size before sending to the backend.
multipart/form-data → /api/predict
2
Preprocessing
PIL opens the image, resizes to 224×224 pixels, converts to RGB, applies MobileNetV2 preprocessing normalisation (pixel values scaled to [−1, 1]).
PIL → tf.keras.applications.mobilenet_v2.preprocess_input
3
MobileNetV2 Inference
The preprocessed tensor is passed through the fine-tuned MobileNetV2 model. Softmax output produces a 38-dimensional probability vector. Top-3 classes and their scores are extracted.
model.predict() → softmax[38]
4
Grad-CAM Generation
Gradients of the top class score with respect to the last convolutional layer (Conv_1) are computed. Global average pooling of these gradients weights the feature maps. The result is upsampled and overlaid on the original image.
GradientTape → Conv_1 → overlay_base64
5
Advisory Lookup
The predicted disease label is matched against the structured advisory database, returning symptom descriptions, treatment protocols, and prevention guidelines.
disease_label → advisory JSON
6
JSON Response
A structured response containing prediction, top_predictions, advisory, xai (overlay base64 + metadata), and model metadata is returned to the frontend in a single request.
{ prediction, advisory, top_predictions, xai, metadata }
